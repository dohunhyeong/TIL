# 🚀 ChromaDB: 핵심 아키텍처 및 필터링 마스터 가이드

ChromaDB는 AI 기반 애플리케이션에 최적화된 **벡터 데이터베이스**입니다. 비구조화된 데이터의 의미를 파악하는 '벡터 검색'과 구조화된 조건을 적용하는 '메타데이터 필터링'을 결합하여 정교한 검색 환경을 제공합니다.

---

## 1. ChromaDB 핵심 개념 및 아키텍처

ChromaDB는 단순히 텍스트를 저장하는 것이 아니라, 데이터의 의미적 유사성을 계산하고 관리합니다.

### 🏗️ 주요 특징
* **고성능 엔진**: 핵심 코어가 **Rust**로 작성되어 Python 기반 DB보다 3~5배 빠른 처리 속도를 자랑합니다.
* **효율적 검색 (HNSW)**: '계층적 탐색형 작은 세상(Hierarchical Navigable Small World)' 알고리즘을 사용하여 수백만 개의 데이터 속에서도 가장 유사한 항목을 즉시 찾아냅니다.
* **유연한 배포 모드**:
    * **클라이언트-서버**: Docker 등을 이용해 독립된 서버로 운영 (HTTP 통신).
    * **스탠드얼론 (Standalone)**: Python 스크립트 내에서 라이브러리처럼 즉시 실행.
* **멀티모달 지원**: 텍스트뿐만 아니라 이미지, 오디오 등 다양한 형태의 데이터를 통합 관리할 수 있습니다.

### 🔄 데이터 워크플로우

1. **컬렉션 생성**: 데이터를 담을 논리적 그룹(테이블)을 만듭니다.
2. **임베딩 및 저장**: 텍스트를 벡터로 변환(Embedding)하여 문서, 메타데이터와 함께 저장합니다.
3. **쿼리 및 필터링**: 유사도 검색과 필터링 조건을 조합하여 최적의 결과를 추출합니다.

## 2. 필터링 시스템 (Filtering)

ChromaDB는 기존 SQL의 `WHERE` 절과 유사하지만, 벡터 검색과 통합되어 더욱 강력한 두 가지 필터링 방식을 제공합니다.

### 🏷️ 메타데이터 필터링 (`where`)
문서와 함께 저장된 부가 정보(태그, 날짜, 작성자 등)를 기준으로 검색 범위를 제한합니다.

| 연산자 | 설명 | 예시 |
| :--- | :--- | :--- |
| `$eq` / `$ne` | 같음 / 같지 않음 | `{"category": {"$eq": "news"}}` |
| `$gt` / `$gte` | 초과 / 이상 | `{"version": {"$gte": 2.0}}` |
| `$lt` / `$lte` | 미만 / 이하 | `{"date": {"$lt": 20240101}}` |
| `$in` / `$nin` | 리스트에 포함 / 미포함 | `{"tag": {"$in": ["AI", "ML"]}}` |

### 📝 문서 필터링 (`where_document`)
문서의 **실제 본문 내용**에 특정 키워드가 포함되어 있는지 확인합니다. (Full-text Search 연계)

* **`$contains`**: 특정 문자열 포함
* **`$not_contains`**: 특정 문자열 미포함
* **주의**: 대소문자를 엄격히 구분합니다.

## 3. 코드 실전 예제

메타데이터 필터와 문서 필터를 조합하여 사용하는 가장 일반적인 형태의 코드입니다.

```python
import chromadb

# 1. 클라이언트 및 컬렉션 설정
client = chromadb.Client()
collection = client.get_or_create_collection(name="my_docs")

# 2. 복합 쿼리 수행
# 조건: 버전이 0.1보다 크고, 본문에 'Python' 또는 'LangChain'이 포함된 문서
results = collection.query(
    query_texts=["AI 기반 프레임워크"],
    n_results=5,
    where={"version": {"$gt": 0.1}},
    where_document={
        "$or": [
            {"$contains": "Python"},
            {"$contains": "LangChain"}
        ]
    }
)
